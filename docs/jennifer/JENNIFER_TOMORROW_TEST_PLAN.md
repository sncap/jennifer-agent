# Jennifer Tomorrow Active Test Plan (D+1)

## 목표

내일부터 적극 테스트를 시작하되, 실패를 빠르게 수집/복구/개선하는 운영 루프로 진행.

## 테스트 원칙

- 실패를 숨기지 않고 즉시 기록
- 같은 실패 2회 반복 시 즉시 원인분석 티켓 생성
- 복구 명령으로 정상화 가능한지 반드시 검증

## 테스트 세션 구성

### Session 1 (오전): 대화/명령 안정성

- 10개 지시 테스트 (단순 4, 중간 4, 복합 2)
- 성공률 목표: 90%+

### Session 2 (오후): 도구/실행 신뢰성

- 편집/커밋/푸시 포함 작업 5건
- 성공률 목표: 100% (부분실패 시 원인과 재시도 기록)

### Session 3 (저녁): 복구/운영 연속성

- reset/new/recovery 각각 1회 이상
- 복구 후 즉시 실작업 1건 성공 확인

## 수집 메트릭

- 지시 성공률
- 재시도율
- 복구 성공률
- 평균 응답 지연(체감)
- 사용자 수정 지시 빈도
