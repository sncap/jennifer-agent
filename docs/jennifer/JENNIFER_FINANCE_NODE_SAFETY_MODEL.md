# Jennifer Finance Node Safety Model (Draft v0.1)

## 목표

주인님의 자산 증식 목표를 지원하되, 자동화 리스크를 통제하는 **승인형 금융 코파일럿** 구조를 정의한다.

---

## 1) 역할 분리

### A. Signal Node (분석 노드)

- 시장/뉴스/실적/수급 데이터 분석
- 매수/매도 후보와 근거 제시
- 실행 권한 없음

### B. Risk Node (리스크 노드)

- 포지션/변동성/손실 한도 감시
- 위반 시 실행 차단 신호 발행
- 보호 규칙 우선권 보유

### C. Execution Node (격리 실행 노드)

- 승인된 주문만 실행
- API 권한 최소화(거래만, 출금 금지)
- 화이트리스트 종목/한도/유효시간 검증

### D. Jennifer Orchestrator

- 전체 컨텍스트 취합
- 최종 승인 요청서 생성
- 실행 후 결과/감사로그 보고

---

## 2) 공동관리(Dual Control)

고위험 액션은 아래 2중 승인 필요:

1. Jennifer의 정책 승인(규칙/리스크 통과)
2. 주인님 최종 승인

고위험 액션 예시:

- 신규 진입
- 포지션 비중 확대
- 손절 무시/완화
- 장 마감 이후 특이 시간대 주문

---

## 3) 제한 위임 원칙

자동 실행은 아래를 모두 만족할 때만 허용:

- 주문 유효시간 (TTL) 내 승인
- 종목/시장 화이트리스트 일치
- 주문 금액 한도 이내
- 일일 손실 한도 미초과
- 전체 익스포저 한도 이내
- 리스크 노드 차단 신호 없음

---

## 4) 운영 모드

- Advisory Mode: 분석/제안만
- Guarded Auto Mode: 제한 위임 자동 실행
- Emergency Halt Mode: 모든 신규 주문 중단

초기 운영은 Advisory → Guarded Auto 순으로 단계적 전환.

---

## 5) 감사/보안

모든 주문에 대해 기록:

- 제안 시각/근거
- 승인자/승인 시각
- 실행 결과/체결 정보
- 사후 평가(의도 대비 결과)

보안 필수:

- 거래 전용 API 키
- 키는 시크릿 스토어 보관
- 노드별 권한 분리
- 비정상 탐지 시 즉시 kill-switch

---

## 6) 현실적 기대치

Jennifer는 수익을 보장하지 않는다.
목표는:

- 대응 공백(업무중/수면중) 최소화
- 감정 개입 감소
- 리스크 규칙 일관 적용
- 장기 성과 개선 확률 증가
